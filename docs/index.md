## Recommended GitHub Pages setup

To publish this directory as a public documentation site:

1. Open the repository on GitHub.
2. Go to **Settings → Pages**.
3. Under **Build and deployment**, choose **Deploy from a branch**.
4. Select the default branch and the `/docs` folder.
5. Save the settings.

> GitHub Pages publishes the selected branch and folder to a public website. Do not put secrets, credentials, private datasets, or internal-only notes in this directory.


## Configuración recomendada de GitHub Pages

Para publicar este directorio como sitio público de documentación:

1. Abre el repositorio en GitHub.
2. Ve a **Settings → Pages**.
3. En **Build and deployment**, elige **Deploy from a branch**.
4. Selecciona la rama principal y la carpeta `/docs`.
5. Guarda la configuración.

> GitHub Pages publica la rama y carpeta seleccionadas como un sitio público. No pongas secretos, credenciales, datasets privados ni notas internas en este directorio.
